# 15. 문제 발굴 (AI BM 특화 리스트업)

> 깔때기 **입구의 입구**. 아이디어가 아니라 *미해결 문제*를 모으는 양식. `/문제발굴 "<영역>"`으로 자동 채우거나 직접 채운다.
> 목적은 **"AI를 핵심 기능으로, 적은 예산으로 BM을 확보할 수 있는 서비스"**의 씨앗 문제를 거르는 것.
> 여기서 점수 높은 문제 → [10_아이디어풀의 문제 인박스](../../10_아이디어풀/문제_인박스.md)에 적재 → 끌리면 한 줄 아이디어로 → `/아이디어검증`.
> 📖 모르는 용어 → [[용어집]]. **BM**=어떻게 돈 버나, **WTP**=지불 의향(실제 쓴 돈으로 봄), **피처 리스크**=그냥 챗GPT 한 번에 풀려 제품이 안 되는 위험, **해자(Moat)**=경쟁자가 못 따라오게 막는 방어막, **에이전트급**=챗 한 번으론 안 되고 여러 단계·도구·자동반복이 필요한 수준.

## A. 어디서 줍나 — 소스 채널 체크리스트

문제는 "생각해내는" 게 아니라 사람들이 **이미 불평하거나 돈을 쓰고 있는 곳**에서 줍는다. 신호 강도 순.

### 🟢 이미 돈을 내고 있는 신호 (가장 강함 — WTP 검증됨)
- ⬜ **크몽·숨고**의 IT·자동화·데이터입력 외주 요청 — 기업이 돈 주고 맡기는 자잘한 반복작업 = AI 대체 1순위
- ⬜ **Upwork / Fiverr** "data entry", "scraping", "automation" 잡 포스팅
- ⬜ **앱스토어/플레이스토어 유료 앱의 ★1~2 리뷰** — 돈 냈는데 화난 사람 = 미충족 수요
- ⬜ **크라우드펀딩(와디즈·킥스타터) 실패작** — 수요는 있었는데 실행이 틀린 것

### 🟡 불만이 텍스트로 쌓이는 곳 (중간 — 수요 신호, WTP는 미검증)
- ⬜ **디스콰이엇(Disquiet) / Indie Hackers** — "이 기능 왜 없냐" 피드백 스레드
- ⬜ **Reddit** (r/mildlyinfuriating, 각 직군 서브레딧), **네이버 카페·지식인**
- ⬜ 대형 SaaS(Notion·Slack·Salesforce 등) 커뮤니티의 **"마지막 1마일" 불편** 글

### 🟠 반복되는 문의/상담 게시판 (반복 빈도 = ① 반복성 증거, WTP는 미검증)
> *답이 달려도 같은 질문이 반복*되는 글만 줍는다 — 검색으로 해결됐으면 반복 안 된다. 반복 빈도는 **①반복성·②데이터비대칭의 공짜 증거**.
- ⬜ **공공기관(규제·강제지출 직결, 최강 광맥)**: 국세청 국세상담 상담사례·국민신문고·건보공단/국민연금 Q&A·금감원 e-민원·한국소비자원 피해구제 사례·법제처 생활법령 백문백답
- ⬜ **민간 종합 Q&A**: 네이버 지식iN(채택 0·같은 질문 반복), [클리앙 "아무거나질문"](https://www.clien.net/service/board/kin)(IT·디지털·구매조언 편향 — 얼리어답터 신호 강·전문직 백오피스는 약), 카페 FAQ 고정글
- ⚠️ **반복 ≠ 통과**: 반복 질문 다수는 *$20 챗이 즉답하는 단순 FAQ*다. 빈도는 ①에만 주고, **⑥은 더 깐깐하게**(개인 서류 검증·멀티스텝·최신 규정 대조라 챗이 못 푸는 반복만 통과). 무료 질문이라 ⑦BM도 후하게 주지 않는다.

### 🔵 검색 의도 / How-to 질문 (사람들이 답을 못 찾은 곳)
- ⬜ Google/네이버 **"왜 ~ 안 되지", "~ 대안", "~ 자동화"** 자동완성·연관검색어
- ⬜ AnswerThePublic / 커뮤니티 **"~하는 법 아시는 분?"** 패턴 (= 검색으로 해결 못 함 = 미해결 수요)
- ⬜ **YC Requests for Startups** — 글로벌 톱 액셀러레이터가 지정한 "나오면 투자" 문제

### ⚫ 거시·공공 (BM 작아지기 쉬움 — 보조로만)
- ⬜ data.go.kr 인기 데이터셋, 정부·기관 백서의 "한계점/과제" 섹션, UN SDGs

### 🔴 본인·실사용자 (가장 차별화되는 원천)
- ⬜ **내가 직접 매주 겪는 페인** (인디해커 1원칙) — 도메인 지식·동기가 따라옴
- ⬜ **실제 사용자 1:1 대화** (→ `/고객인터뷰`)

> 한 영역을 정하면 위 채널을 **여러 갈래로** 훑는다(한 각도로는 다 못 본다). 출처 링크는 반드시 남긴다.

## B. 어떻게 거르나 — 문제점수 + 피처 리스크 게이트

AI로 *적은 예산에* BM을 만들려면 문제 자체가 특정 성질을 가져야 한다. 5개는 **문제점수(0~2)**, 1개는 **게이트(통과/탈락)**, 1개는 **비즈니스 점수(0~2)**다. 비즈니스 가치는 이제 게이트가 아니라 점수로만 본다.

### 점수축 (각 0~2, 합 /10) — 높을수록 좋은 문제
| 축 | 핵심 질문 | 0 / 1 / 2 |
|---|---|---|
| **① 반복성** (High Frequency) | 매일/매주 반복되는데 자동화 안 된 일인가? | 드묾 / 가끔 / 매주+ |
| **② 데이터 비대칭** (Data Asymmetry) | 정보는 있는데 흩어져 있어 일반인이 못 찾/분석하나? | 아님 / 약간 / 심함 |
| **③ 비용 장벽 대중화** (Cost Barrier) | 전에는 전문가(변호사·세무사·디자이너)가 필요해 일반인이 포기하던 일인가? | 아님 / 일부 / 명확 |
| **④ Why now** | AI로 *방금* 풀 수 있게 됐나, 원래도 됐나? | 원래 됨 / 애매 / 최근 가능 |
| **⑤ 방어가능성**(Moat) | 내 도메인지식·데이터·유통채널이 있나, 누구나 만드나? | 누구나 / 약간 / 내 강점 |

### 게이트 (탈락이면 인박스에 ⚠️ 표시, 함부로 승급 금지)
| 게이트 | 질문 | 통과 / 탈락 |
|---|---|---|
| **⑥ 피처 리스크 — "20달러 챗 테스트"** | 이 문제가 **$20 구독 챗(Claude/ChatGPT)에 복붙 한 번**으로 풀리나? | 챗 한 번에 풀림 = **탈락(피처)** / **에이전트급**이라야 풀림 = **통과(제품)** |

### 비즈니스 점수 (0~2, 게이트 아님)
| 축 | 핵심 질문 | 0 / 1 / 2 |
|---|---|---|
| **⑦ BM/WTP 점수** | 누군가 *이미* 돈·시간을 쓰고 있나(외주·유료앱·수작업)? | 없음/막연한 호감 / 시간·수작업·간접비용은 씀 / 직접 지불·반복 구매·전문가 상담 강함 |

> **읽는 법**: 문제점수 8~10 + ⑥ 통과 = 강력 후보. ⑦BM/WTP는 승급 차단이 아니라 우선순위와 다음 검증의 초점을 정하는 점수다. 예: `8점 ⑥✅ ⑦BM=0`이면 문제는 좋지만 비즈니스 검증 선결, `8점 ⑥✅ ⑦BM=2`이면 바로 검증행 후보.

> 🎯 **능력 대역(⑥의 핵심)**: 적은 예산 AI 서비스가 살 수 있는 구역은 좁다.
> - **$20 챗으로 풀림** → 피처. 플랫폼(GPT·Gemini)이 곧 삼킨다 → ⑥ **탈락**.
> - **$100 Code-Max 에이전트급** (멀티스텝·툴 사용·코드 실행·사적 데이터 RAG·검증 루프·시스템 연동) → **스윗스팟**. 프롬프트·통합·자기 데이터로 만들 수 있어 *적은 예산에 가능* → ⑥ **통과**.
> - **새 모델 능력·학습이 필요** → 연구·대자본 영역, 적은 예산 불가 → 보류.
> *"Code Max는 되는데 $20 챗은 안 되는" 문제만 노린다.* 단장 추출(명세서 1장)은 챗이 하지만, *대량+검증+시스템 연동+매일 자동*은 에이전트라야 한다 — **제품은 챗 라인 위에서 산다.**

> ⚠️ **승급 전 경쟁 리얼리티 체크 (8점 후보 필수)**: 표면 검색은 **⑤해자·⑥피처리스크를 후하게** 매긴다. 승급(→백로그) 직전, 8점 후보에 한해 `<문제 핵심> 솔루션/앱`·`ChatGPT로 <문제> 해결`을 **1~2회만** 더 검색해 본다. **직접 경쟁 전용 제품**이 나오면 ⑤를 0~1로, **범용 LLM이 무료로** 하면 ⑥을 탈락으로 내려 재채점한다. ⑦BM/WTP가 낮으면 탈락시키지 말고 "비즈니스 검증 선결"로 표시한다. (깊은 경쟁조사는 `/아이디어검증`이 한다 — 여긴 가짜 8점만 싸게 거른다.)

## C. 무엇을 적나 — 인덱스 한 줄 + 상세 카드

문제 1개 = **① 인덱스 표 1행 + ② 상세 카드 1개**. (인박스가 [한눈에 인덱스] + [상세(영역별) 카드] 2단 구조라 둘 다 적는다.) 단순 나열 금지 — **AI API로 바로 프로토타입 검증 가능한 형태**로.

**① 인덱스 표(스캔용, 5칸)** — `## 📥 발굴된 문제 — 한눈에` 표에 한 줄 추가:

| 컬럼 | 무엇을 |
|---|---|
| **#** | 번호(⑥ 게이트 탈락이면 `⚠️#N`) |
| **문제 (요약)** | 문제 한 줄을 ~44자로 압축(마크다운·링크 제거) |
| **영역** | 발굴 영역(예: 건기식·금융·규제검수) |
| **점수** | 문제점수 합(정정 시 정정값, 0~10). 필요하면 `8/BM2`처럼 ⑦BM 점수 병기 |
| **상태** | 🔴버림 / 🟡관찰 / 🟢승급 / 🟠Pivot |

**② 상세 카드** — `## 📥 상세 (영역별)`의 해당 영역(`> 영역:…` 또는 `### 헤더`) 아래에 카드 추가:

```
#### #N · {점수}점 · {상태}
> {문제 한 줄 — 누가/언제/무엇 때문에 고통받나}

- **출처** 채널·글/리뷰 + 링크
- **현재 워크어라운드** 지금 어떻게 때우나(엑셀 노가다/카톡 수소문/외주)
- **AI 핵심기능** LLM(요약·변환)/RAG(문서기반)/Vision(이미지) 중 무엇
- **데이터확보** 공공데이터·크롤링 무료 가능? (Low/Med/High)
- **점수** `2/1/2/2/1=8 ⑥✅ ⑦BM=2` (①②③④⑤ 문제점수 + ⑥ 피처 게이트 + ⑦BM/WTP 점수)
- **BM 가설** 구독(SaaS)/건당 과금/B2B 중 무엇
- **다음 액션** 버림 / 더 관찰 / **백로그 한 줄로 승급**
```

✅ 자가검수
- [ ] 각 문제에 **출처 링크**가 있나 (머릿속 추측 아닌가)
- [ ] **현재 워크어라운드**가 적혔나 (없으면 진짜 문제인지 의심)
- [ ] ⑥(피처 리스크)을 **정직하게** 봤나 (점수만 높고 피처인 함정 거름)
- [ ] ⑦BM/WTP 점수를 후하게 주지 않았나 ("있으면 좋겠다"와 실제 돈·시간 지출을 구분)
- [ ] 승급한 문제는 [한 줄 아이디어 양식](01_아이디어한줄.md)으로 옮길 수 있나
