# 11. 아이디어 검증 (문제정의 → 경쟁 → 문제·쐐기 → BM점수)

> 새 아이디어 1개를 한 번에 검증하는 산출물. `/아이디어검증 "<아이디어>"`로 자동 생성하거나 직접 채운다.
> **이게 10_아이디어풀 → 20_탐색중 승급 게이트.** 판정이 Go면 같은 폴더에서 페르소나·JTBD로 더 파고, Drop이면 99_아카이브로.
> 📖 모르는 용어(워크어라운드·JTBD·해자·BM·Why now·시장공백 등) → [[용어집]].

---

> **아이디어**: 〔한 줄〕
> v0.1 · YYYY-MM-DD · 담당 @이니셜
> **판정**: ⬜ Go ⬜ Pivot ⬜ Drop
> **BM/WTP 점수**: ⬜ 0 ⬜ 1 ⬜ 2

## 1. 문제 정의 (이 검증의 기준점)
모든 §2·§3 판단은 여기에 근거한다. 근거 없으면 "가정"이라 표시.

- **한 문장 문제**: 〔누가 / 언제 / 무엇 때문에 / 지금은 어떻게 때우는지〕
- **페르소나**: 〔구체 1명 — 이름·나이·상황〕
- **JTBD**: 〔~할 때, ~하고 싶다, 그래서 ~할 수 있게〕
- **빈도·강도**: 〔얼마나 자주 / 얼마나 아픈가 — 돈·시간·감정〕
- **현재 대안(워크어라운드)**: 〔지금 사람들이 실제로 쓰는 것 = 진짜 1차 경쟁자〕
- **핵심 통찰 / Why now**: 〔왜 지금 풀리나 — 기술·규제·행동 변화〕

## 2. 유사 해결방식 탐색 (제품 + 유튜브)
> §1의 "현재 대안"에서 출발해 찾는다. 추측 말고 **검색으로 확인**, 출처 링크 남길 것.

### 2a. 유사·경쟁 제품
| 제품 | 누구를 위해 | 핵심 기능 | BM/가격 | 강점 | **이 문제에서 빈 곳** |
|---|---|---|---|---|---|
| | | | | | |

### 2b. 유튜브 — 유사 해결방식·수요 신호
> "이 문제 이렇게 해결" 류(자작·DIY·꿀팁·튜토리얼·앱/제품 리뷰) 영상을 찾는다. 조회수·댓글 = 수요 신호. 조회수는 확인되면 적고, 아니면 비운다(지어내지 말 것).

| 영상(링크) | 채널 | 조회수 | 제안하는 해결방식 | 신호(수요/워크어라운드/경쟁) |
|---|---|---|---|---|
| | | | | |

- **유튜브가 말해주는 것**: 〔사람들이 지금 실제로 쓰는 워크어라운드 / 수요가 뜨거운가 / 이미 영상으로 데모되는 대체 솔루션〕

### 종합
- **시장 공백(White space)**: 〔제품·영상 통틀어 문제 정의의 어느 조각을 아무도 안 풀고 있나 = 우리의 쐐기〕
- **차별화 한 줄**: 〔기존 대안 대비 우리가 *유일하게* 잘하는 것〕

## 3. 문제·쐐기 검증 (판정의 핵심)
| 질문 | 판단 | 근거(§1·§2 연결) |
|---|---|---|
| **수요** — 문제가 자주·아픈가? | 강/중/약 | |
| **차별화** — §2 공백이 진짜 쐐기인가? | 강/중/약 | |
| **시장 규모** — 누가 몇 명, 대략 크기 | | (방향성 추정) |
| **해자** — 시간이 지나며 강해지나? | 강/중/약 | |
| **피처 리스크** — 20달러 챗/플랫폼/기존 대안이 삼키나? | 통과/탈락 | |
| **리스크** — 규제·신뢰·실행 난이도 | 상/중/하 | |

## 4. BM/WTP 점수 (비즈니스 가치 — 게이트 아님)
> 돈을 낼 가능성은 **판정 게이트가 아니라 점수**로 본다. 낮다고 바로 Drop하지 말고, 다음 검증의 초점으로 넘긴다.

| 항목 | 점수/판단 | 근거(§1·§2 연결) |
|---|---|---|
| **⑦BM/WTP 점수** | 0 / 1 / 2 | 0=돈 낼 신호 없음 · 1=시간/수작업/간접비용은 있음 · 2=외주/유료앱/전문가 상담/반복 구매 강함 |
| **누가 돈 내나** | B2C / B2B / B2B2C / 후행수익 | |
| **수익 가설** | 직접결제 / 구독 / 건당 / 제휴 / 데이터 / 광고 / 미정 | |
| **비즈니스 검증 필요도** | 낮음/중간/높음 | BM/WTP 0~1이면 보통 높음 |

## 5. 현실 점검 체크리스트 (판정 전 점검)
> §1~§4에 근거해 정직하게 체크. **빈칸 하나 = 리스크 하나.** 막연하면 ⬜로 두고 §6 "다음 검증"에 넣는다. 03·07은 BM/WTP 점수 근거이지 자동 Drop 게이트가 아니다.

- ⬜ **01** 이 문제를 겪는 사람 이름 3명을 지금 댈 수 있나요 — 〔이름/관계〕
- ⬜ **02** 그 사람이 지금 이 문제를 어떻게 대충 해결하고 있는지 한 문장으로 쓸 수 있나요 — 〔§1 워크어라운드와 연결〕
- ⬜ **03** 그 사람이 이 문제 때문에 실제로 돈이나 시간을 쓰고 있나요 — 〔무엇에 얼마 / §4 BM점수 근거〕
- ⬜ **04** 경쟁 서비스를 3개 이상 찾아 써 봤나요 — 〔§2a에서 직접 써 본 것〕
- ⬜ **05** 우리만의 "왜 지금, 왜 우리"가 한 문장으로 나오나요 — 〔§2 종합 차별화와 연결〕
- ⬜ **06** 첫 1000명 사용자가 어디서 들어올지 구체적인 채널이 있나요 — 〔채널/근거〕
- ⬜ **07** 이 서비스가 한 달에 1만 원 받을 만한 가치를 주고 있나요 — 〔지불의향 근거 / 낮으면 비즈니스 검증 선결〕

## 6. 판정 & 다음 한 수
- **판정**: Go / Pivot / Drop — 이유(문제·쐐기 관점에 근거해 1~2줄)
  - **Go**: 문제·쐐기·초기 사용자 가설이 선명하다. BM/WTP가 낮아도 "비즈니스 검증 선결"로 Go 가능.
  - **Pivot**: 문제는 진짜지만 타깃·쐐기·사용맥락이 틀렸거나 좁혀야 한다.
  - **Drop**: 문제가 가짜이거나, 쐐기가 없거나, 20달러 챗/플랫폼/기존 대안에 흡수되어 제품이 되기 어렵다. **BM/WTP가 낮다는 이유만으로 Drop하지 않는다.**
- **BM/WTP 점수**: 0 / 1 / 2 — 〔§4 요약〕
- **가장 위험한 가정**: 〔틀리면 전부 무너지는 단 하나〕
- **다음 검증**: 〔그 가정을 *싸게* 확인할 방법 — 인터뷰 N명 / 랜딩페이지 / 수동 컨시어지〕
  - 위 체크리스트에서 **⬜로 남은 항목을 가장 위험한 가정 후보로** 다룬다.
  - BM/WTP 0~1이면 다음 검증에 **비즈니스 검증 선결**을 반드시 포함한다.
  - 인터뷰로 검증한다면 → `/고객인터뷰`로 The Mom Test 기반 가이드(`인터뷰_가이드.md`) 생성.
