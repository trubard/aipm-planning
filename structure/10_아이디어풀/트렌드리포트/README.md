# 트렌드 리포트 모음

> 문제 발굴의 **원재료** 창고. [문제 인박스](../문제_인박스.md) *바로 옆*에 둔다 — 리포트를 모아두고, `/문제발굴`이 영역을 훑을 때 검색 신호와 **교차 대조**하는 근거로 쓴다.
> 리포트 자체는 "문제"가 아니다. 여기 쌓인 트렌드에서 *미해결 문제 후보*를 뽑아 문제점수·피처 리스크 게이트로 태깅한 결과가 [문제 인박스](../문제_인박스.md)에 적재된다.

## 무엇을 넣나
- 시장·세대·산업 트렌드 리포트 (예: 오픈서베이 Z세대 트렌드, 업종별 시장 보고서)
- 소비/행동 데이터, 설문 결과, 업계 동향 PDF·요약
- "Why now"(④)의 근거가 되는 변화 신호 — 규제 변화, 신기술 가용성, 비용 구조 변화

## 어떻게 정리하나
- 파일 하나 = 리포트 하나. 이름은 `YYYY-MM-DD_출처_제목` 권장 (예: `2025-01_오픈서베이_Z세대트렌드2025.md`).
- PDF 원문은 그대로 두되, **핵심 3~5줄 요약 + 출처 링크**를 같은 이름의 `.md`로 남기면 `/문제발굴`이 읽기 쉽다.
- 리포트에서 문제 후보를 뽑아 인박스에 적었다면, 그 행에 출처로 이 리포트를 링크해 추적성을 남긴다.

## ⚠️ 배포 정책 (저작권)
- 리포트 **원문은 공유 레포에 올리지 않는다**. 저작권 이슈로 `.gitignore`에서 이 폴더는 `README.md`만 추적한다.
- 즉 여기 쌓는 PDF·요약 `.md`는 전부 **로컬 전용**이다. 공유가 필요하면 출처 링크만 인박스에 남긴다.
