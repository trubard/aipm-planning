---
description: structure/ 골격에서 my-idea/ 작업 공간을 생성하거나, my-idea/를 빈 상태로 리셋한다
---

당신은 이 워크스페이스의 관리자다. 아래 인자에 따라 **내 작업 공간 `my-idea/`를 생성하거나 초기화**한다. `structure/`(시스템 골격)는 절대 건드리지 않는다.

> **불변 원칙**: 복제 원본은 항상 `structure/`다. `my-idea/`만 만들거나 비운다. **삭제는 되돌릴 수 없으므로** 리셋 전에는 반드시 사용자 확인을 받는다.
> **쉬운 말 규칙**: 사용자에게 보고할 때 전문용어를 피하고 쉬운 말로 쓴다. 처음이라 용어가 낯설어 보이면 `structure/90_템플릿/용어집.md`를 안내한다.

## 인자
$ARGUMENTS

- 비어 있으면: 아래 "상태 감지"로 `my-idea/` 유무를 보고 **무엇을 할지 사용자에게 먼저 묻는다**(생성 / 리셋 / 취소).
- `생성` 또는 `init`: `my-idea/`가 없을 때만 골격을 복제한다. 이미 있으면 덮어쓰지 말고 현황만 보고.
- `리셋` 또는 `reset`: `my-idea/`의 모든 산출물을 지우고 빈 골격으로 되돌린다. **파괴적** — 실행 전 확인 필수.

## 상태 감지 (반드시 먼저)
1. `my-idea/`가 있는지, 있다면 그 안에 산출물이 얼마나 있는지(아이디어 폴더 수·파일 수) 센다.
2. `structure/`에 복제 원본 4개 폴더(`10_아이디어풀`·`20_탐색중`·`30_검증정의`·`99_아카이브`)가 있는지 확인한다. 없으면 멈추고 사용자에게 알린다(시스템 손상).

## 할 일

### A. 생성 (my-idea/ 없음)
- 다음을 실행해 골격을 복제한다(`90_템플릿`은 복제하지 않는다 — 양식은 `structure/`에서 읽는다):
  ```bash
  mkdir -p my-idea && cp -R structure/10_아이디어풀 structure/20_탐색중 structure/30_검증정의 structure/99_아카이브 my-idea/
  ```
- 복제 직후 `my-idea/`가 `.gitignore`로 무시되는지 `git check-ignore my-idea/`로 확인한다.

### B. 리셋 (my-idea/ 있음)
1. **확인** — 지워질 내용을 구체적으로 보여준다: 아이디어 폴더 N개·파일 M개. "정말 비울까요? 되돌릴 수 없습니다"라고 묻고 **명시적 동의(예/yes)** 를 받기 전엔 실행하지 않는다. 거부하면 취소.
2. (선택) 사용자가 백업을 원하면 `my-idea/`를 `_백업/my-idea_<날짜>/`로 복사한 뒤 진행한다(`_백업/`은 `.gitignore` 대상이어야 함 — 아니면 먼저 추가 안내).
3. **리셋 실행**:
   ```bash
   rm -rf my-idea && mkdir -p my-idea && cp -R structure/10_아이디어풀 structure/20_탐색중 structure/30_검증정의 structure/99_아카이브 my-idea/
   ```

## 보고
끝나면 2~3줄로: ① 한 동작(생성/리셋/취소) ② `my-idea/`의 현재 상태(폴더·파일 수, 비었는지) ③ 다음 한 수(예: `/아이디어검증`으로 시작, 또는 백로그에 한 줄 적기).
