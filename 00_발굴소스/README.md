# 00_발굴소스 — 아이디어 원천

> 깔때기의 맨 위. **"사람의 반복되는 불편"** 신호를 여기서 줍는다 → 한 줄로 좁혀지면 [../10_아이디어풀/아이디어_백로그.md](../10_아이디어풀/아이디어_백로그.md)로.

## 폴더
| 폴더 | 용도 |
|---|---|
| [검색트렌드/](검색트렌드/) | 수집한 검색 키워드 아카이브 (`YYYY-MM-DD 검색트렌드.md`). 작성·수집법은 [검색트렌드 작성법.md](검색트렌드%20작성법.md) |
| [리서치/](리서치/) | 리서치 결과·자료 정리 |
| [인박스/](인박스/) | 일단 떠오르는 거 아무거나 던져두는 곳 |

## Claude로 소스 채우기
이 폴더는 Claude가 직접 읽고 쓸 수 있다. 예:
- "오늘 검색 트렌드 수집해서 `00_발굴소스/검색트렌드`에 노트로 저장해줘"
- "이 리서치 자료 요약해서 `00_발굴소스/리서치`에 넣어줘"

## 원석 → 아이디어로 넘기기
소스를 본 뒤 "이건 제품 신호다" 싶은 인사이트를 짧게 정제하고, 끝에 **"→ 누구의 / 어떤 반복 불편인가"** 한 줄을 붙인다. 그게 곧 [아이디어 백로그](../10_아이디어풀/아이디어_백로그.md) 한 줄의 씨앗.
