# AIPM — 팀 아이디어 발굴 워크스페이스

> v1.1 · 2026-06-16 · 이 저장소는 **팀이 함께 새 제품 아이디어를 발굴 → 탐색 → 검증 → 정의**로 흘려보내는 깔때기(funnel)다.
> 모든 아이디어는 아래 단계 폴더를 **왼쪽 → 오른쪽**으로 통과한다. 막히면 `99_아카이브`로.
>
> 👉 **처음이면 [START_HERE.md](START_HERE.md)** · 협업 규칙은 [운영규칙.md](운영규칙.md).

## 단계별 깔때기

```
00_발굴소스 ──► 10_아이디어풀 ──► 20_탐색중 ──► 30_검증정의 ──► (출시/이관)
  트렌드·리서치     한 줄 아이디어     문제정의·페르소나   PRD·MVP·피치덱
  (vault 연동)      백로그 1줄         JTBD·Journey·HMW    NorthStar·Aha·AARRR
                                          │
                                          └──► 99_아카이브 (드롭/보류)
```

| 폴더 | 무엇이 들어가나 | 다음 단계로 가는 게이트 |
|---|---|---|
| **00_발굴소스** | 검색 트렌드, 리서치, 인박스 원석 | 신호가 "사람의 반복되는 불편"으로 좁혀지면 → 한 줄 아이디어로 등록 |
| **10_아이디어풀** | `아이디어_백로그.md` 한 곳에 **한 줄 아이디어**만. 가볍게 많이 | 사용자·대안·핵심 한 가지가 또렷하면 → 탐색 폴더 생성 |
| **20_탐색중** | 아이디어당 폴더 1개. 문제정의·페르소나·JTBD·Journey·HMW | 문제가 진짜고 크고, HMW에서 쓸 만한 솔루션이 나오면 → 검증·정의 |
| **30_검증정의** | 아이디어당 폴더 1개. NorthStar·Aha·AARRR·MVP 스펙·PRD·피치덱 | MVP 가설 검증되면 → 본 개발/스핀아웃 이관 |
| **90_템플릿** | `기획_산출물_템플릿.md`(작성 가이드) + `양식/`(빈 양식 11종) | — |
| **99_아카이브** | 드롭·보류된 아이디어 (왜 접었는지 한 줄 남기고 이동) | 부활 시 다시 10/20으로 |

## 지금 들어있는 아이디어

_아직 없음._ 깔때기가 비어 있는 초기 상태입니다 — 첫 아이디어를 [10_아이디어풀/아이디어_백로그.md](10_아이디어풀/아이디어_백로그.md)에 한 줄로 적는 것부터 시작하세요.

## 새 아이디어 시작하는 법
1. [00_발굴소스/](00_발굴소스/)에서 트렌드·리서치를 보며 **반복되는 불편**을 찾는다
2. [10_아이디어풀/아이디어_백로그.md](10_아이디어풀/아이디어_백로그.md)에 **한 줄**로 적는다 (양식은 백로그 상단).
3. 끌리면 `20_탐색중/<아이디어명>/` 폴더를 만들고 [90_템플릿/기획_산출물_템플릿.md](90_템플릿/기획_산출물_템플릿.md) 순서대로 채운다.
