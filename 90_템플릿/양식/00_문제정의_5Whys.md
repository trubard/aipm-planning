# 5 Whys 문제 정의 — [주제]

> 작성법·예시: [[기획_산출물_템플릿#0. 5 Whys 문제 정의]]
> 규칙: 사람 탓("게을러서")에서 멈추지 말고 구조/도구/프로세스 결함까지. 원인이 여럿이면 갈래로 분기.

**증상 (처음 보이는 문제)**:

- **Why 1** 왜? →
- **Why 2** 왜? →
- **Why 3** 왜? →
- **Why 4** 왜? →
- **Why 5** 왜? →

**∴ 재정의된 문제** (근본 원인을 푸는 한 문장):

> → 이 문장이 [[01_아이디어한줄]]과 [[10_PRD_1페이저]]의 문제 정의 입력이 된다.

## 자가 검수
- [ ] 증상이 구체적인가 (누가·언제·무엇을)
- [ ] 각 Why가 앞 답을 받아 이어지나 (점프 없이)
- [ ] 사람 탓이 아니라 구조/도구/프로세스 결함까지 내려갔나
- [ ] 멈춘 지점이 우리가 손댈 수 있는 원인인가
- [ ] 원인이 여럿이면 분기로 다뤘나
- [ ] 재정의된 문제가 처음 증상과 다른 문장인가
- [ ] 1번 아이디어 한 줄로 자연스럽게 이어지나
