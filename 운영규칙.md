# 운영 규칙 (팀 발굴 워크스페이스)

> 누가 써도 같은 품질, 서로 안 부딪히게. 짧게 유지한다. 시작은 [START_HERE.md](START_HERE.md).

## 1. 깔때기 단계와 승급 게이트

| 단계 | 폴더 | 들어있는 것 | **다음 단계로 가는 게이트** |
|---|---|---|---|
| 발굴 | `00_발굴소스` | 트렌드·리서치·인박스 원석 | 신호가 "사람의 반복되는 불편"으로 좁혀짐 |
| 풀 | `10_아이디어풀` | 한 줄 아이디어 (백로그 1곳) | 사용자·대안·핵심 한 가지·결과가 또렷 |
| 탐색 | `20_탐색중/<아이디어>` | 문제정의·페르소나·JTBD·Journey·HMW | 문제가 진짜·크고, HMW에서 쓸 솔루션 도출 |
| 검증·정의 | `30_검증정의/<아이디어>` | NorthStar·Aha·AARRR·MVP·PRD·피치덱 | MVP 가설 검증됨 → 본개발/스핀아웃 이관 |
| 보관 | `99_아카이브` | 드롭·보류 | (부활 시 10/20으로) |

**승급·드롭 결정**: 담당자가 제안 → 주간 리뷰(또는 PR 리뷰)에서 팀이 확정. 혼자 30단계로 직행 금지.

## 2. 한 아이디어 = 한 폴더 = 한 담당자(driver)
- `20_탐색중/`, `30_검증정의/` 아래 **아이디어명 폴더**를 만든 사람이 담당자.
- 폴더명: 한글 아이디어명 (예: `장보기도우미`). 공백 대신 붙여쓰기.
- 담당자/단계는 [10_아이디어풀/아이디어_백로그.md](10_아이디어풀/아이디어_백로그.md) 표에서 한눈에 관리한다.
- **남의 아이디어 폴더 수정**은 직접 말고 PR 또는 코멘트로. 자기 폴더는 자유.

## 3. 한 줄 아이디어 양식 (10_아이디어풀)
> **[누가—구체조건]**가 **[지금 하는 대안/습관]** 대신 **[핵심 한 가지]**로 **[결과—가능하면 숫자]**

- 원석은 양식 안 맞아도 OK, 일단 적는다.
- 끝에 작성자·날짜 태그: `— @이니셜 2026-06-16`
- 트렌드/리서치에서 나온 건 출처 한 줄 같이.

## 4. 산출물 작성
- 형식·순서·체크리스트는 [90_템플릿/기획_산출물_템플릿.md](90_템플릿/기획_산출물_템플릿.md).
- 빈 양식은 [90_템플릿/양식/](90_템플릿/양식/) (5Whys·페르소나·JTBD·HMW·NorthStar·Aha·AARRR·PRD 등).
- 같은 아이디어 안에서는 **사용자 이름·숫자를 고정**(예: 사용자=김OO 34세 / 대상=박OO 67세)해 문서 간 일관 유지.

## 5. 파일·협업 컨벤션
- 파일명: `산출물종류_아이디어명.md` 또는 양식 번호 접두사. 날짜는 frontmatter나 문서 상단에.
- GitHub: 새 아이디어 탐색은 가능하면 **브랜치 → PR**. 백로그 한 줄 추가 정도는 바로 커밋 OK.
- 문서 상단에 `> vX.Y · 날짜 · 담당자` 한 줄 메타를 유지한다.
